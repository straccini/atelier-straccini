## Modifiche previste al backend

### A) Nuovo endpoint `transcribe` in `doPost`

Input atteso nel body:

```json
{
  "action": "transcribe",
  "target": "medolce" | "ricettario",
  "text": "testo libero incollato dall'utente",
  "hints": { "category": "...", "source": "..." }
}
```

Output atteso: oggetto strutturato conforme al target (schema `medolce` o `ricettario`), validato e **non ancora salvato**. Il salvataggio avviene con una chiamata successiva `action: "add"` dopo conferma utente.

Motore AI candidato: **Gemini via Apps Script `UrlFetchApp`** (opzione preferita: chiave in `PropertiesService`, niente credenziali in chiaro). Alternative: Vertex AI, OpenAI — vincoli di policy e costi discussi in `docs/frontend/trascrizione-ai-spec.md`.

### B) Nuovo routing `addRicettario`

Speculare a `addMedolce`, mappa lo schema di output NLM (o della trascrizione AI) sullo schema `ricettario`:

| Campo NLM / AI        | Campo Ricettario  |
|-----------------------|-------------------|
| `name`                | `name`            |
| `category`            | `category`        |
| `emoji`               | `emoji`           |
| `difficulty`          | `difficulty`      |
| `prep_time`           | `prep`            |
| `cook_time`           | `cook`            |
| `servings`            | `servings`        |
| `ingredients[]`       | `ingredients` (JSON) |
| `steps[]`             | `steps` (JSON)    |
| `notes`               | `notes`           |
| `tags[]`              | `tags` (JSON)     |
| `photo_url`           | `fotoUrl`         |
| `audio_url`           | `audioUrl`        |
| `nlm_source_url`      | `sourceUrl`       |
| `createdAt`           | `createdAt`       |

### C) Nuovo endpoint `bulkImportFromNlm`

Per import massivi dalle fonti NLM, con deduplica automatica basata su `sourceUrl` + hash del titolo normalizzato.

```json
{
  "action": "bulkImportFromNlm",
  "target": "ricettario",
  "records": [ { ...ricetta1 }, { ...ricetta2 } ],
  "dedupe": true
}
```

Risposta: `{ added, updated, skipped, errors }`.

## Criteri di accettazione

1. Trascrizione AI produce oggetti conformi agli schemi `HEADERS.medolce` e `HEADERS.ricettario` definiti in v3.2 (nessun campo mancante o extra non previsto).
2. `addRicettario` non introduce regressioni su `addMedolce`.
3. `bulkImportFromNlm` deduplica correttamente su fonte+titolo e riporta tutti i conflitti in `errors`.
4. Tutti i nuovi endpoint rispettano il contratto di risposta `{ ok, status, ... }` introdotto in v3.2.
5. Il frontend (v3.3 UI) chiama i nuovi endpoint senza rompere i flussi precedenti.
